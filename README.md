# Predicting Replication from Domain-Stripped Fingerprints, Where Citations Fail (reproduction package)

Can you tell, from a paper's text alone, whether its finding will survive replication? Citation impact
cannot: on 481 replicated psychology findings (FORRT Replication Database), field-weighted citation
impact predicts replication at chance (AUROC 0.473). A **domain-stripped multi-lens fingerprint** of the
abstract (one cached LLM call per paper, four lenses: computational, experimental, finding, qualitative)
read by plain bag-of-words Naive Bayes does much better, and beats the reproducible baseline of the
state-of-the-art method class on every dataset.

**Reproduction runs offline: no LLM, no API key, no network.** The fingerprints and all cached metadata
are bundled under `data/`, so every number reproduces from the shipped files (`scikit-learn` for the
core; `sentence-transformers` only for the novelty axis and the dense-embedder ablation cells).

## The headline (Table 1: aggregated AUROC from abstracts, 5-fold CV)

| dataset | TF-IDF+LR (SOTA method class, reproduced) | raw abstract BoW+NB | **multi-lens fingerprint (ours)** |
|---|---|---|---|
| FORRT (481) | 0.630 | 0.658 | **0.701** (+0.072 vs TF-IDF, p=0.002) |
| Yang/Uzzi raw (259) | 0.635 | 0.619 | **0.747** (+0.112, p<0.001) |
| Yang/Uzzi cleaned (254) | 0.618 | 0.615 | **0.720** (+0.102, p<0.001) |

Companion results, all reproduced by shipped scripts: citations at chance (FWCI AUROC 0.473, AP 0.496);
cross-dataset generalization (train FORRT, test Yang/Uzzi: 0.706, overlap removed); novelty and
verifiability are orthogonal axes (r = -0.05, TOST p = 0.016); stacking novelty onto the replication
predictor *hurts* (0.701 -> 0.668, 0.720 -> 0.678, CIs exclude zero); the fingerprint novelty converges
with an independent reference-spread novelty (Shibayama; Spearman rho = +0.156, p = 0.0005, n = 489).

## Quickstart

Python 3.12+ (developed and verified on 3.14; `.tool-versions` pins the verified interpreter for
mise/asdf users, but any recent CPython works).

```
make setup        # one time: create .venv and install requirements
make reproduce    # every paper number + the three figures
make quick        # embedder-free core only (Table 1, orthogonality, figures, P@k)
```

or without make:

```
pip install -r requirements.txt
python reproduce.py            # add --quick for the embedder-free core
```

The first full run downloads two small sentence-transformer checkpoints (all-MiniLM-L6-v2,
all-mpnet-base-v2) from HuggingFace for the novelty axis and two ablation cells; everything else needs
only `scikit-learn` over the bundled files.

## Where each paper number comes from

Every number in the paper is regenerated by a shipped script (run any of them alone with
`PYTHONPATH=src python src/<script>.py` from the package root):

| paper item | script |
|---|---|
| Table 1 (multi-lens fingerprint vs fixed baselines) + AP + full-text row | `experiment_maintable.py` |
| Orthogonality TOST, cross-dataset 0.706/0.708, lens complementarity | `experiment_revisions8.py` |
| Figures 1-3 + citations-at-chance (FWCI 0.473 / AP 0.496) | `make_figures.py` |
| Novelty-hurts stacking (0.701 -> 0.668; 0.720 -> 0.678) + vocabulary counts | `experiment_revisions6.py` |
| Table 2 cells (representation x reader) | BoW + MiniLM: `experiment_sota_match.py`; TF-IDF: `experiment_revisions3.py`; all-mpnet: `experiment_revisions5.py` |
| Table 3 (novelty vs reference-corpus size) | `experiment_revisions2.py` (CIs: `revisions3`, Bonferroni: `revisions5`) |
| Table 4 (quadrant replication rates) + multiplicative-vs-additive tau | `experiment_revisions3.py` |
| Shibayama reference-spread convergence (rho +0.156) + nearest-cited-prior null | `compute_ref_novelty2.py` |
| ICLR expert-novelty leg (rho 0.115, AUROC 0.588) | `experiment_novelty_expert2.py` |
| Composite ranking P@k (verifiability 0.80 vs product 0.60 at k=10) | `experiment_composite_pak.py` |
| Psych-background orthogonality (disclosed alternative) | `experiment_revisions4.py` (TOST: `revisions5`) |
| Abstract-vs-full-text contrasts ("what carries the signal") | `experiment_perlens.py`, `experiment_threefold.py` (need the rebuilt full-text corpus, see below) |
| Novelty/verifiability axes (`data/value_scores.csv`, `data/rnd_novelty.csv`) | `experiment_value.py`, `experiment_rnd.py` (bundled precomputed) |

The four verbatim lens distillation prompts (Claude 3 Haiku, temperature 0) are in `SUPPLEMENTARY.md`;
the fingerprints they produced are bundled under `data/fingerprints*/`, so no LLM is needed.

`src/` also keeps the exploratory scripts from the research process (earlier experiment rounds,
alternative framings) for full transparency; the table above is the canonical map for the paper's
numbers.

## What is bundled, and redistribution

This package ships **derived artifacts and openly-licensed metadata only**:

- the **fingerprints**: domain-stripped LLM re-descriptions of each abstract (four lenses). They do not
  reproduce the source text.
- **abstracts and bibliographic metadata** from OpenAlex (CC0; abstracts reconstructed from the
  inverted index), the public FORRT Replication Database export, and the released Yang/Uzzi and
  Altmejd OSF files.
- **No full text is redistributed.** The abstract-vs-full-text contrast experiments need a locally
  rebuilt full-text corpus: `data/fulltext_manifest.csv` records the DOIs/URLs, and the harvest
  scripts (`src/harvest_oa.py` for open access; `src/browser_*.py` for subscription access through
  your own institution, credentials via `.env`, see `example.env`) rebuild it. All headline numbers
  are abstract-only and need none of this.

## Integrity

```
sha256sum -c SHA256SUMS
```

Expected outputs: the paper's three figures are regenerated as `fig1_scatter.pdf`, `fig2_lenses.pdf`,
`fig3_roc.pdf`; every stage prints its numbers to stdout under a banner naming the paper item.
